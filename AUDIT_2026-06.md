# nexus-zim — Audit Notes (June 2026)

Part of the tri-repo ecosystem audit. The full cross-app analysis (integration
map, federation security model, findings table) lives in
`scholastic-services/TRI_REPO_AUDIT_2026-06.md`.

## Role
Inter-school **sports** platform (handball + netball): competitions, fixtures,
live scoring, standings, and Scholastic-card athlete verification. Runs on its
own Supabase project (`cnfrf…`) and talks to the Scholastic hub via the
`scholastic-{federation,push,sync}` and `verify-card` edge functions.

## Health (this session)
- `tsc --noEmit -p tsconfig.app.json` → **0 errors**
- `vite build` → **pass**
- `vitest` → **15 tests pass** (was a single placeholder)

## Changes
- Added real unit coverage for the in-scope-sport filter (`nexusSports.ts`) and
  the schools constants/`tierLabel` helper (`schools.ts`). These guard the
  handball/netball scoping that gates cross-discipline data pulled from the
  shared schema.

## Verified correct (no change needed)
- `verify-card` properly authenticates the caller (Supabase role
  `hic`/`admin`/`super_admin`), forwards to the Scholastic bridge with HMAC s2s
  + federation JWT, and logs each attempt with `jti`/`nonce` for anti-replay.
- The `.or()` filters in `scholastic-federation` / `scholastic-push` interpolate
  internally-resolved UUIDs (not request input), so they are not injectable.
